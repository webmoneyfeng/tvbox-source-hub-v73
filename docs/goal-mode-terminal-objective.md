# Codex 目标模式终局目标提示词

## 当前终局目标

```text
请在每完成一个阶段任务的规划之后，继续结合上下文并严格遵照《终局全局流程图》规划下一阶段执行计划。

每一阶段都必须承接前一阶段结果，并说明下一阶段如何承接；规划粒度必须从终局到全局、局部、节点、末梢逐层展开，再反向核对是否服务终局。

终局目标：
把 tvbox-source-hub-v73 打造成“用户喜欢的、0 投诉的、可商业化收费的顶级 TVBOX 点播 + 直播源”。

换句话说，终局不是“技术上能用”，而是“用户愿意长期订阅、愿意推荐、使用中没有有效投诉”的商业级 TVBOX 源服务。

目标模式进行中大目标提示词：
请在每完成一个阶段任务的规划之后，继续结合上下文并严格遵照《终局全局全景流程图》规划下一阶段执行计划；每个阶段都必须承接前一阶段结果，并说明下一阶段如何承接；规划粒度必须从终局到全局、局部、节点、末梢逐层展开，再反向核对是否服务终局；循环执行“规划 → 实现 → 审计 → 修复 → 验证 → 下一阶段规划”，直到达成：用户喜欢的、0 投诉的、可商业化收费的顶级 TVBOX/FongMi/影视仓点播 + 直播源。

核心验收：
1. TVBox/FongMi/影视仓电视端功能达到商业级体验，不只是堪用，而是用户喜欢用、愿意持续用。
2. 直播 + 点播都可用、稳定、干净、高清、可搜索、可筛选、可播放。
3. 全量实时 / 准实时更新，允许小级别时间差，但用户端必须无感更新。
4. 任意分类、筛选、搜索、详情、播放线路都按按钮语义正确返回。
5. 同一节目或视频在任意分类、筛选、搜索结果中不重复。
6. 搜索支持片名、别名、主演、年份、地区、类型、内容形态、标签等多维召回。
7. 源发现、源准入、覆盖审计、语义审计、遥控器全链路验证形成自动闭环。
8. 可以面向用户收费运营，但自运营基础设施坚持免费优先：不买主机、不买付费 CDN、不代理大规模视频流、不把付费/登录/私有 token 源作为主链路。
9. 商业化销售的是配置维护、聚合体验、稳定性和自动化运维能力，不托管视频文件。
10. “0 投诉”是第一性验收：不能导入、打不开、搜不到、空按钮、重复、广告、卡顿、更新慢、分类错、播放失效、文案混乱、成人/直播/点播边界混乱等都必须进入根因审计，不能用“技术上已通”代替用户满意。
11. 同一套维护链路必须同时发布两个点播接口：全量版保留成人内容，洁净版去掉成人内容；两个接口必须同时间生成、同时间发布、同时间审计，不允许后续维护只更新其中一个。
12. 免费优先更新目标必须持续收敛用户可感知滞后：Cloudflare Cron 热源探测目标 15 分钟级，热点快照目标 30 分钟级，全量快照目标 2 小时级；可见更新时间必须来自验证通过的快照或验证通过的热源探测，失败刷新不能污染电视端更新时间。

执行方法：
按“设计 → 实现 → 审计 → 修复 → 验证 → 下一阶段规划”的循环推进，直到终局目标达成。
任何阶段不能只做局部修补；发现问题必须追根因，归类为源覆盖、解析逻辑、快照命中、搜索召回、去重、播放、缓存、并发、免费额度或电视端请求差异。

阶段性完成标准：
每阶段结束必须交付：
1. 当前阶段完成了什么。
2. 证据文件、命令输出或线上接口证明。
3. 未完成项与根因。
4. 下一阶段从终局到末梢的执行计划。
```

## 执行守恒规则

- 终局不变：用户喜欢 + 0 投诉 + 可商业化收费 + 实时更新 + 点播直播双可用 + 自运营全免费 + TV 端顶级体验。
- 架构不临时化：可以阶段实施，不能阶段发明架构。
- 免费优先不等于功能降级：先用缓存、KV、静态快照、调度降频、索引裁剪控制成本，再考虑功能取舍。
- 遥控器验收不是“能点就算通”，而是响应格式、数量、内容语义、分页、详情、播放、缓存一致性都符合预期。
- 空结果不是直接删除按钮，必须根因追踪后决定修逻辑、补解析、扩源、降级隐藏或标记物理限制。
